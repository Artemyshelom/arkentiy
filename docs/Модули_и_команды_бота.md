# Аркентий — Модули и команды бота

> Справочник для настройки доступов в `/доступ` и понимания что умеет бот.

---

## Как это работает

Каждому чату или пользователю в `/доступ` можно выдать **набор модулей**.  
Модуль = набор команд/функций. Нет модуля → бот молча игнорирует команды из этого чата.

**Приоритет проверки прав:**  
`admin → зарегистрированный чат → зарегистрированный пользователь → .env fallback`

Если чат зарегистрирован → его права применяются, права пользователя внутри чата игнорируются.

**DM (личка):** только admin (255968113). Остальные — молча игнорируются.

## Модули

| Модуль | Метка в /доступ | Что даёт |
|--------|----------------|----------|
| `late_alerts` | 🔴 Алерты | **Push-уведомления** об опозданиях (пороги: +15 🟡, +30 🟠, +45 🔴🔴). Команд нет — бот сам шлёт при событии. Управляется `/тишина`. |
| `late_queries` | 📋 Опоздания | Ручные запросы опозданий в реальном времени и за день |
| `search` | 🔍 Поиск | Поиск заказов по номеру или телефону |
| `reports` | 📊 Отчёты | Статус точек в реалтайме + исторические отчёты за любой период |
| `marketing` | 📈 Выгрузка данных | Экспорт данных в Google Sheets |
| `finance` | 💰 Финансы | Банковские выписки (разбивка + Сбер-сверка) + реестр ТБанк (трекер онлайн-оплат) |
| `audit` | 🔍 Аудит | Ежедневный отчёт о подозрительных операциях (сторно + ручная скидка, отмены с суммой) |
| `admin` | 🛠 Админ | Управление доступами и конкуренты |

## Команды по модулям

### 🔴 late_alerts — Алерты
Не команда. Бот автоматически отправляет в чат сообщение при опоздании заказа.  
**Три уровня:** 🟡 +15 мин → 🟠 +30 мин ‼️ → 🔴🔴 +45 мин КРИТИЧНО  
Чтобы работало: чат зарегистрирован + включён `late_alerts` + выбраны города.

| Команда | Что делает |
|---------|-----------|
| `/тишина [длительность]` | Заглушить алерты в этом чате (напр. `/тишина 30`, `/тишина 1ч`, `/тишина 2 часа`). Макс 2 ч. |
| `/тишина` | Показать статус — активна ли тишина и сколько ещё |

Каждое использование `/тишина` логируется в БД.

### 📋 late_queries — Опоздания
| Команда | Что делает |
|---------|-----------|
| `/опоздания [фильтр]` | **Реалтайм** — активные опоздавшие доставки прямо сейчас (из in-memory). Фильтр: город или точка. |
| `/опоздания [фильтр] [дата]` | **История** — опоздания за день из БД, группировка по точкам. |
| `/самовывоз [фильтр]` | **Реалтайм** — активные опоздавшие самовывозы (статус: готовится/приготовлен). |

### 🔍 search — Поиск
| Команда | Что делает |
|---------|-----------|
| `/поиск [номер или телефон]` | Карточка заказа по номеру или телефону клиента |
| `/поиск 12345 томск` | Поиск с фильтром по городу/филиалу (пересечение с правами чата) |

### 📊 reports — Отчёты
| Команда | Что делает |
|---------|-----------|
| `/статус` | Текущая выручка, чеки, скидки, времена доставки по точкам |
| `/отчёт [филиал] [период]` | Полный отчёт за любой период из БД. Примеры ниже. |
| `/точные [филиал] [дата]` | Список заказов на точное время (предзаказы) |
| `/повара` | Повара на сегодняшней смене |
| `/курьеры` | Состояние доставки (на линии, опаздывающие) |

**Примеры `/отчёт`:**
- `/отчёт` — все точки за вчера
- `/отчёт барнаул` — Барнаул за вчера (агрегат + кнопки филиалов)
- `/отчёт барнаул 14.02` — Барнаул за 14 февраля
- `/отчёт барнаул неделя` — Барнаул за прошлую календарную неделю (пн-вс)
- `/отчёт барнаул эта_неделя` — с понедельника по вчера
- `/отчёт томск месяц` — Томск за 30 дней
- `/отчёт 01.02-15.02` — все точки за диапазон
- `/отчёт февраль` — все точки за февраль

**Что показывает:** выручка, чеки, средний чек, скидки (с разбивкой по типам и количеству), SailPlay, себестоимость, опоздания, средние времена (готовка, ожидание курьера, в пути, итого), количество точных заказов, штат за день (только в однодневных отчётах).

### 💰 finance — Финансы

**Банковская выписка (Сбер):** отправь `.txt` файл из СберБизнес (формат 1С) → разбивка по точкам + сверка Сбер-эквайринга с iiko + вшивка комиссии.

**Реестр ТБанк:** отправь `.xlsx` ежедневного отчёта ТБанк → сверка онлайн-оплат по номерам заказов. Stateful: бот запоминает заказы и алертит, если оплата не приходит > 4 дней.

Команд нет — автодетект по содержимому файла.

**Что возвращает (выписка Сбер):**
- Сводку: приход/расход по каждой точке, эквайринг Сбер (gross до комиссии)
- 8 файлов (по одному на р/с): Барнаул-1..4, Абакан-1..2, Черногорск-1, Агрегаторы

**Что возвращает (реестр ТБанк):**
- Подтверждённые оплаты по каждой точке
- Расхождения сумм (ТБанк vs iiko)
- Просроченные оплаты (ожидают > 4 дней)

### 🔍 audit — Аудит
Ежедневный отчёт в 09:27 в чаты с включённым модулем.  
Что отслеживается:
- **Сторно + ручная скидка** — сторнирование чека с последующей ручной скидкой (время, тип оплаты, тип скидки)
- **Отмены с суммой** — отменённые заказы ≥ 500₽ (тип оплаты, причина отмены)
- **Отмены с причиной** — любая отмена ≥ 200₽ с указанной причиной

| Команда | Что делает |
|---------|-----------|
| `/аудит` | Подозрительные операции за вчера по городу чата |
| `/аудит Томск` | То же, Томск |
| `/аудит Томск 21.02` | Томск, 21 февраля |

Подключить: `/доступ` → чат → включить модуль **🔍 Аудит** + выбрать город(а).

## Что показывают реалтайм-команды

**Источник:** in-memory `_states` — данные обновляются каждые 30 сек из Events API.

**`/опоздания`** (доставка):
- Заказы курьером у которых `сейчас > plan_time`
- Сортировка: сначала самые долгие
- Инфо: опоздание (мин), номер, точка, имя клиента, телефон, план время, статус, курьер (если в пути)

**`/самовывоз`** (самовывоз):
- Аналогично, только самовывозные заказы (без курьера)
- Статусы: готовится / приготовлен
- Инфо без курьера

## Настройка чата для алертов

Чтобы группа получала алерты об опозданиях:
1. `/доступ` → Чаты → найти нужный чат
2. Включить модуль `🔴 Алерты`
3. Выбрать города (какие точки мониторить)

Если чат не зарегистрирован — `/доступ` → Добавить чат → ввести chat_id группы.
