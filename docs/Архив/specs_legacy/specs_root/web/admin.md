> Статус: ✅ Реализовано — 26 февраля 2026
> Автор: @веб
> Дата: 26 февраля 2026

---

## Страница

**Название:** Суперадмин-панель (Super-admin Panel)

**URL:** `/admin` — список тенантов; `/admin/tenants/{id}` — карточка тенанта; `/admin/monitor` — мониторинг

**Тип:**
- [x] Страница с данными + управление
- [x] Приватная (нужен JWT с ролью `superadmin`)

---

## Авторизация

**Нужна:** да, JWT с дополнительным claim `role: superadmin`

**Механизм:** JWT в httpOnly cookie, тот же механизм что у клиентского кабинета

**Если нет токена:** редирект на `/login.html?redirect=/admin`

**Если токен есть, но роль не superadmin:** 403 Forbidden, редирект на `/cabinet`

**Доступ:** только Артемий (255968113). В JWT payload: `{ "user_id": ..., "tenant_id": null, "role": "superadmin" }`

---

## Аудитория

Только суперадмин (Артемий). Управляет всеми тенантами: просматривает, настраивает, решает проблемы. Заходит чтобы:
- Зарегистрировать нового клиента
- Настроить клиенту чаты, модули, точки
- Посмотреть статус всех подключений сразу
- Разобраться с проблемой конкретного клиента

**Устройство:** desktop-only (операционный инструмент, не нужен мобайл).

## Стек и ограничения

**Фронтенд:** HTML + Tailwind + Vanilla JS (как остальные страницы проекта)

**Адаптивность:** desktop-only (операционный инструмент для суперадмина, мобайл не нужен)

**Браузеры:** Chrome (последняя версия)

**URL-схема:**
- `/admin` → список тенантов
- `/admin/tenants/new` → создание тенанта
- `/admin/tenants/{id}` → карточка тенанта (вкладки через URL hash: `#subscription`, `#chats`, `#branches`, `#billing`, `#users`)
- `/admin/monitor` → мониторинг

---

## Мультиклиентность

- [x] Суперадмин видит данные **всех** тенантов
- Все `/api/admin/*` endpoints проверяют `role == "superadmin"` в JWT
- Обычный клиентский JWT (role=tenant) → 403 на всех `/api/admin/*`

## Зависимости от @интегратора

### Новые API endpoints:

| Endpoint | Метод | Описание |
|----------|-------|----------|
| `/api/admin/tenants` | GET | Список всех тенантов со статусами |
| `/api/admin/tenants` | POST | Создать нового тенанта |
| `/api/admin/tenants/{id}` | GET | Все данные конкретного тенанта |
| `/api/admin/tenants/{id}` | PUT | Обновить профиль тенанта |
| `/api/admin/tenants/{id}/subscription` | PUT | Изменить подписку тенанта |
| `/api/admin/tenants/{id}/chats` | GET | Список чатов тенанта |
| `/api/admin/tenants/{id}/chats` | POST | Добавить чат тенанту |
| `/api/admin/tenants/{id}/chats/{chat_id}` | PUT | Обновить чат |
| `/api/admin/tenants/{id}/chats/{chat_id}` | DELETE | Удалить чат |
| `/api/admin/tenants/{id}/branches` | GET | Список точек тенанта |
| `/api/admin/tenants/{id}/branches` | POST | Добавить точку |
| `/api/admin/tenants/{id}/branches/{bid}` | DELETE | Удалить точку |
| `/api/admin/tenants/{id}/users` | GET | Пользователи тенанта |
| `/api/admin/tenants/{id}/users` | POST | Добавить пользователя |
| `/api/admin/tenants/{id}/users/{uid}` | DELETE | Удалить пользователя |
| `/api/admin/tenants/{id}/iiko/test` | POST | Проверить iiko тенанта |
| `/api/admin/monitor` | GET | Статус всех подключений |

### Авторизация:
- Middleware `require_superadmin` — отдельная проверка от обычного `require_auth`
- JWT payload: `{ "user_id": ..., "role": "superadmin" }` (без tenant_id)

### Файлы фронтенда:
- `web/admin/index.html` — список тенантов
- `web/admin/tenant.html` — карточка тенанта (с вкладками)
- `web/admin/new.html` — создание тенанта
- `web/admin/monitor.html` — мониторинг
- `web/js/admin.js` — общая логика (sidebar, auth check, API helper для /api/admin/*)

## Заметки для @интегратора

- Все `/api/admin/*` — 403 если `jwt.role != "superadmin"`
- При создании тенанта (`POST /api/admin/tenants`) → автоматически создать запись в `tenants`, `tenant_users` (admin), запустить trial
- `temp_password` в ответе на создание — одноразовый, клиент меняет при первом входе
- iiko test для конкретного тенанта: читать credentials из таблицы `tenant_iiko_config`
- Мониторинг: читать из кэша (таблица `connection_health`), не делать live-запросы на каждый GET `/api/admin/monitor`
- Таблица `connection_health`: `tenant_id`, `type` (iiko/telegram), `chat_id` (nullable), `status`, `checked_at`, `response_ms`
