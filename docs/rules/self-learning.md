# Самообучение после задачи

После каждой выполненной задачи — в таком порядке:

1. **Журнал.md** (`docs/Журнал.md`) — что сделано, технически (для разработчика)
2. **Уроки_и_баги.md** — если нашли новый баг или антипаттерн
3. **CHANGELOG.md** (`docs/CHANGELOG.md`) — на языке пользователя (для сотрудников и клиентов)

## Правила обновления CHANGELOG.md

**Добавлять:** новая команда/модуль, изменение поведения, заметное пользователю изменение.

**Не добавлять:** рефакторинг без изменения поведения, скрытые баги, технические изменения (БД, конфиги).

**Как писать:**
- Язык: что **получает** пользователь, не что изменилось в коде
- Без: `_states`, `dict`, `PostgreSQL`, `in-memory`, `async`, `DATABASE_URL`
- С: примеры команд в backtick-формате, конкретные сценарии
- Новая запись — сверху файла, после шапки

**Шаблон:**
```
## ДД месяца ГГГГ — суть одной фразой

Одно предложение — суть для тех кто не следит за деталями.

**`/новая_команда [аргумент]`** — что делает

Описание без технятины. Что видит пользователь.

`/пример использования Барнаул`
```
