## ПРИНЦИП ВНЕДРЕНИЯ

При автоматизации любой ручной работы — **не говорить сотрудникам** сразу. Пусть продолжают делать вручную. Параллельно ИИ делает то же самое. Сравниваем скорость и точность → считаем окупаемость → потом принимаем решение об изменении позиции.

## ЭТАП 5 — SaaS-упаковка

### Онбординг-воронка

```
Регистрация (TG бот) → Конфигуратор (точки + модули) → Цена + оплата (ЮKassa) → Ввод данных iiko → Тест коннекта → Автоактивация (доступы в TG группах)
```

| # | Задача | Описание |
|---|--------|----------|
| S.1 | **Онбординг визард** | Пошаговый TG-бот: iiko URL → API key → выбор точек → тест → активация |
| S.2 | **ЮKassa** | Создание счёта, вебхук подтверждения, рекуррентные платежи |
| S.3 | **Health monitoring** | Алерт при падении iiko API или задержке данных — тебе и клиенту. Клиент видит статус "все подключения работают" |
| S.4 | **Управление подпиской** | Добавить модуль, отключить, 7 дней grace period, данные хранятся 90 дней |
| S.5 | **Уведомления** | -3 дня до продления, день X, +3 "просрочка", +7 "отключение" |
| S.6 | **Метрики продукта** | MAU, retention, какие модули используют чаще, где отваливаются |
| S.7 | **DPA / юридика** | Договор на обработку данных до первого платного клиента |

### Ценообразование

| Тариф | Цена | Что включено |
|-------|------|-------------|
| Базовый (до 5 точек) | 15-20К/мес | Core: статусы, опоздания, алерты, отчёты |
| Базовый (каждая точка) | 10К/мес | Для больших сетей |
| Расширенный (+LLM, +финансы) | 25-35К/мес | Всё из базового + аудитор + ABC + рекомендации |
| Подключение (разово) | 30-50К | Настройка, интеграция, онбординг |
| Превышение токенов LLM | по факту | При превышении лимита из подписки |

> При 100 точках на базовом = 1-2 млн/мес MRR

## ⚠️ ИЗВЕСТНЫЕ УЗКИЕ МЕСТА

| Проблема | Суть | Решение |
|----------|------|---------|
| **OLAP-пресеты** | Для OLAP каждый клиент должен иметь пресеты в своём iiko BO. Создаются вручную, нельзя автоматизировать через API | Инструкция + XML-шаблон для импорта; или ограничить функционал данными из Events API (пресеты не нужны) |
| **PostgreSQL** | SQLite не выдержит параллельные запросы от нескольких tenants | Миграция в Месяц 1, блокер для SaaS |
| **DPA** | Обработка финансовых и персональных данных клиентов | Договор до первого платного клиента |
| **Поддержка** | При 10+ клиентах поддержка съест всё время | FAQ-бот / база знаний заранее |

## ТЕХНИЧЕСКИЕ ЗАДАНИЯ (АКТУАЛЬНЫЕ)

### Задача 1.7: Недельный отчёт
**Решение:** свободная форма запроса (все данные уже в БД). Данные: выручка, опоздания, топ-точки, топ-курьеры по опозданиям за неделю.  
**Срок:** месяц 2.

### Задача 1.9: Фильтр по городу в /поиск
**Текущее поведение:** `/поиск 199505` → ищет по всем точкам.  
**Решение:** парсить второй аргумент как фильтр: `/поиск 199505 томск`.  
**Алиасы:** `том`, `томск`, `барнаул`, `баул`, `абакан`, `абк`, `чгск`.

### Задача: Бэкфилл смен из OLAP (shifts_raw)
**Контекст:** `shifts_raw` хранит смены с момента запуска. Нужно загрузить историю с 01.06.2025.  
**Статус:** ждём OLAP-пресета от Артемия.

### Задача: PostgreSQL миграция
**Контекст:** SQLite → PostgreSQL, tenant_id во всех таблицах, изоляция данных per tenant.  
**Срок:** Месяц 1. **Блокер для SaaS.**
