## 🎓 11 марта 2026 — Станислав: консультант по системе

**Ново:** второй AI-ассистент для onboarding'а новых управляющих и клиентов.

**Чем помогает:**
- Объясняет команды `/статус`, `/отчёт`, `/опоздания`  
- Рассказывает про нормативы и правильный контроль доставки
- Ищет примеры из рынка доставки еды (тренды, best practices)
- При нужде проверит конкретную цифру из твоей системы для подтверждения совета

**Важно:** Станислав — наставник и учитель. Если нужна регулярная численная аналитика или глубокий разбор — это роль Бориса (другого ассистента).

## 🔧 28 февраля 2026 — Исключение смен оплаты из статистики опозданий

Заказы со сменой типа оплаты больше не портят статистику времени доставки.

**Проблема:** когда оператор меняет тип оплаты (нал↔онлайн), iiko пересоздаёт заказ и доставка формально происходит за 0 минут. Это ломало среднее время и создавало ложные рекорды.

**Решение:** добавлено поле `payment_changed` в таблицу `iiko_bo_events`. Определяется по комментарию «смен» в данных заказа. Такие заказы исключаются из расчёта опозданий.

**Обновлено:** 491 историческая запись.

## 💳 25 февраля 2026 — Сверка онлайн-оплат с ТБанк

Бот теперь умеет сверять реестр ТБанк с данными iiko — по каждому заказу отдельно.

**Как работает:**
Кидаешь xlsx-файл ежедневного отчёта ТБанк в чат → бот автоматически:
- Распознаёт формат реестра
- Парсит транзакции по каждой точке
- Сопоставляет с заказами из iiko по номеру заказа
- Показывает, что подтверждено, что расходится, чего нет в iiko

**Stateful трекер:**
Бот запоминает каждый онлайн-заказ и следит за ним, пока оплата не придёт из реестра ТБанк. Если деньги не приходят дольше 4 дней — алерт.

**Что видно в отчёте:**
- Подтверждённые оплаты по каждой точке
- Расхождения сумм (ТБанк vs iiko)
- Заказы, которых нет в iiko
- Просроченные оплаты (ожидают > 4 дней)
- Комиссия ТБанк по каждой точке

## 📊 25 февраля 2026 — Умные фильтры выгрузки клиентов

Команда `/выгрузка` теперь понимает сложные маркетинговые запросы — сегменты, которые раньше требовали ручной работы в Excel.

**Новые сценарии:**

`/выгрузка Барнаул клиенты делали 2 и более заказа в период ноябрь-январь`
Находит всех клиентов, которые сделали нужное количество заказов именно в указанном периоде.

`/выгрузка новые клиенты ноября, которые до сих пор не сделали повторного заказа`
Выбирает тех, кто попробовал в первый раз — и больше не вернулся. Аудитория для реактивации.

`/выгрузка Томск повторные клиенты с 3 и более заказами за прошлый месяц`
Самые активные гости месяца — база для лояльности и подарков.

`/выгрузка Абакан заказывали август-декабрь, но не заказывали в январе и феврале`
Клиенты, которые были активны, но пропали. Классика win-back.

**Дополнительные фильтры:**
- Тип оплаты: `наличные`, `карта`, `онлайн`
- Источник заказа: `приложение`, `сайт`, `колл-центр`
- Только заказы с жалобами / без жалоб
- Уникальные клиенты без повторов: `список для обзвона`

**В файле теперь два новых столбца:** «Заказов за период» и «Источник заказа».

## 💰 25 февраля 2026 — Банковская выписка: разбивка + сверка с iiko

**Модуль:** обработка банковской выписки из СберБизнес (формат 1С).

**Как работает:**
- Отправь `.txt` файл (выгрузка 1С из СберБизнес) в чат с ботом
- Бот автоматически распознаёт формат, парсит документы, разбивает по расчётным счетам
- Возвращает 8 отдельных файлов (по каждой точке) для загрузки в iiko + сводку

**Сводка включает:**
- Приход / расход по каждой точке с количеством операций
- Эквайринг Сбер: сумма до комиссии по каждой точке
- Контроль целостности (сверка сумм)

**Сверка эквайринга с iiko:**
- Автоматическая сверка сумм Сбер-эквайринга из банка с данными iiko (PayTypes: «Картой при получении» + «Сбербанк»)
- По каждой точке: банк → iiko, комиссия, расхождение
- ✅ при совпадении, ⚠️ при любом расхождении
- Группировка по городам, итоги + суммарная комиссия

**Точки:** Барнаул-1..4, Абакан-1..2, Черногорск-1, Агрегаторы

**Доступ:** модуль `finance` (только admin).

## 🚀 23 февраля 2026 — Опоздания в реальном времени, режим тишины, умные алерты

Большое обновление по блоку опозданий. Главная идея: администратор должен за 5 секунд понять картину — кто сейчас опаздывает, на сколько, где курьер.

### Новое

**`/опоздания [город или точка]`** — активные опоздания прямо сейчас

Раньше эта команда показывала историю за день. Теперь — живой список тех, кто опаздывает в эту минуту. Можно быстро обзвонить клиентов или уточнить у курьера.

Что видно по каждому заказу:
- На сколько минут опаздывает
- Номер заказа и точка
- Имя и телефон клиента
- Плановое время доставки
- Статус: готовится / приготовлен / в пути
- Имя курьера (если в пути)

Фильтр: `/опоздания Барнаул` — только Барнаул. `/опоздания Барнаул 2` — конкретная точка.

---

**`/самовывоз [фильтр]`** — новая команда

То же самое, но для заказов на самовывоз — те, у кого уже прошло плановое время, а они ещё не забрали. Без курьера, только статус приготовления.

---

**`/день [фильтр] [дата]`** — история опозданий за день

Переехало с `/опоздания`. Показывает сводку по всем точкам за выбранный день из базы данных. Удобно смотреть итоги в конце смены или разбирать конкретный день.

Примеры:
- `/день` — сегодня
- `/день вчера` — вчера
- `/день Томск вчера` — Томск, вчера
- `/день 20.02.2026` — конкретная дата

---

**`/тишина [длительность]`** — режим тишины для алертов

Когда пришло много алертов и нужно работать без шума — можно выключить их на время.

Примеры:
- `/тишина 30` — выключить на 30 минут
- `/тишина 1ч` — на час
- `/тишина 2 часа` — максимум
- `/тишина` без аргументов — проверить, активен ли режим и сколько осталось

Важно: тишина только для конкретного чата. Другие группы получают алерты как обычно. Максимум 2 часа.

### Изменения в алертах об опозданиях

Раньше бот присылал одно сообщение при опоздании ≥15 минут. Теперь — три уровня с нарастающей срочностью:

| Опоздание | Сигнал |
|-----------|--------|
| +15 мин | 🟡 Первый сигнал |
| +30 мин | 🟠 Нарастание ‼️ |
| +45 мин | 🔴🔴 КРИТИЧНО |

Каждый уровень приходит один раз. Если включён режим тишины — алерт пропускается, но счётчик двигается: после выхода из тишины старые уровни не всплывают.
