# TG UX Spec — шаблон

> Этот файл заполняет @ux-бот. Интегратор читает его перед реализацией — никакого угадывания.
> Имя файла: `название_функции.md` (например: `courier_filter.md`, `status_command.md`)

---

## Команда / триггер

**Название:** `/команда [аргумент]`

**Тип триггера:**
- [ ] Команда пользователя (`/команда`)
- [ ] Крон (автоматически по расписанию)
- [ ] Загрузка файла (`.xlsx`, `.txt`)
- [ ] Нажатие inline-кнопки
- [ ] Inline-вопрос (бот спрашивает → пользователь отвечает)

**Расписание (если крон):** `09:30 МСК ежедневно`

---

## Условия доступа

**Модуль:** `reports` / `late_alerts` / `finance` / `audit` / `admin` / без модуля

**Где работает:** личка admin / зарегистрированный чат / любой чат

**Фильтр по городу:** да / нет (если да — берётся из прав чата или из аргумента)

## Сообщение

### Основное состояние

```
📊 Статус — Барнаул, 14:32

▸ Основной: 87 400 ₽ · 43 чека
▸ Малиновского: 54 200 ₽ · 27 чеков

⏱ Среднее время доставки: 48 мин
🔴 Опозданий сейчас: 3
```

> Укажи точный текст с плейсхолдерами в `{фигурных скобках}` если значение динамическое.
> Покажи реальный пример с числами — не абстрактные «данные здесь».

### Пустое состояние (нет данных)

```
Нет данных за выбранный период.
```

### Состояние ошибки / недоступности

```
⚠️ Не удалось получить данные. Попробуй через минуту.
```

## Кнопки (inline keyboard)

```
[ Абакан ] [ Томск ] [ Черногорск ]
[ Обновить ] [ За день ]
```

> Опиши раскладку строками. Каждая строка — новый ряд кнопок.
> Укажи что происходит при нажатии каждой кнопки.

| Кнопка | Действие |
|---|---|
| Абакан | Повторить запрос с фильтром Абакан |
| Обновить | Переотправить то же сообщение с актуальными данными |
| За день | Показать итоги за весь день |

---

## Технические ограничения

- Длина сообщения: < 4096 символов (если больше — разбить на части или сделать сводку)
- `parse_mode`: `HTML` / `Markdown` / без форматирования
- Можно редактировать сообщение (`edit_message`) или только новое: да / нет
- Изображения/файлы: нет / да (тип: `.csv`, фото)

---

## Мультиклиентность

- [ ] Функция универсальная (работает для любого клиента)
- [ ] Функция специфична для пилота Ёбидоёби (временно, с пометкой — переделать при масштабировании)

Если универсальная — убедись что города/точки берутся из конфига клиента, а не захардкожены.

## Зависимости (что нужно от @интегратора)

- Нужен новый endpoint / новая таблица / новый job: да / нет
- Если да — описать кратко что именно
