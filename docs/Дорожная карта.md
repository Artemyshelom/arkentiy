## 1) Где мы сейчас

### Сделано

- PostgreSQL в проде, миграция с SQLite выполнена
- Multi-tenant база и управление доступами через `/доступ`
- Multi-tenant бот на одном токене, полная изоляция тенантов
- Core команды TG-бота: `/статус`, `/поиск`, `/аудит`, `/отчёт`, `/выгрузка`
- Финансовый контур: сверка эквайринга и трекер онлайн-оплат (ТБанк)
- Базовый кабинет клиента (MVP): авторизация и ключевые страницы
- Первый SaaS-клиент: Никита Шабуров (Канск, Зеленогорск, Ижевск)

### В работе

- **Быстрые доработки на эту неделю:**
  - Статус кассовой смены в `/статус` (если закрыта — показывать уведомление)
  - Оптимизация получения статуса активных заказов (OLAP vs real-time Events API)
  - Детализация по станциям в `/статус` (готовка, ожидание курьера, курьер)
  - Автоматический забор выписок с почты + присылка в чат
  - Выручка пофилиально в автоматических итогах дня
  - Интерфейс `/статус` под разные часовые пояса + адаптация утреннего отчёта
  - Переработка интерфейса аудита (упрощение, явная маркировка причин отмены)

- Стабилизация качества данных и закрытие техдолга по отчётам/времени
- UX-доработки существующих команд и прав доступа
- Стандартизация документации (журнал, backlog, specs, changelog)

### Ограничения

- OLAP-пресеты у новых клиентов требуют ручной настройки
- Исторические смены по сотрудникам ограничены доступностью API

## 3) План на 90 дней

## Фаза A — Надёжная база

- Деплой-протокол как обязательный стандарт
- Закрытие критичных багов по данным и алертам
- Единый мониторинг состояния jobs и интеграций

## Фаза B — Продукт для пилотов

- 2-3 внешних пилота (кроме Ёбидоёби)
- Онбординг + оплата + базовые SLA
- Обратная связь пилотов и быстрые итерации

## Фаза C — Масштабируемость

- Роли/доступы и изоляция данных per tenant без ручных костылей
- Расширение аналитики и рекомендаций (LLM-модули по необходимости)
- Подготовка к системной монетизации

---

## 4) Коммерческий вектор

- Core тариф: мониторинг + отчёты + алерты
- Pro тариф: финансы + расширенный аудит + интеллектуальные рекомендации
- Отдельно: внедрение/подключение и доработки под клиента

---

## 5) Что не тащим в активный план

- Низкоприоритетные интеграции без влияния на выручку/контроль
- Экспериментальные AI-фичи без понятной бизнес-метрики
- Большие инициативы без ресурса на поддержку после релиза
