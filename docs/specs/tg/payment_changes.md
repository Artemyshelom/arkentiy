# Исключение смен оплаты из статистики

## Проблема

При смене типа оплаты (онлайн → наличные и т.п.) iiko пересоздаёт проводку. Заказ закрывается с временем новой проводки, из-за чего:
- Время доставки = 0 или отрицательное
- Статистика средних времён занижается

Сейчас Света вручную вычленяет такие заказы. Нужно автоматизировать.

## Признаки заказа со сменой оплаты

Любой из:
- Комментарий содержит "смен" (регистронезависимо)
- Ожидание курьера ≥ 120 мин И время доставки = 0

## Что сделать

### 1. Флаг в БД
Добавить поле `payment_changed: bool` в таблицу заказов (или отдельная таблица связей).

### 2. Определение при обработке
При сохранении/обновлении заказа проверять признаки → ставить флаг.

### 3. Исключить из статистики
В расчёте средних времён (`/статус`, `/отчёт`) игнорировать заказы с `payment_changed=True`.

### 4. Показывать счётчик
В отчётах добавить строку:
```
⚠️ Исключено из расчёта: N (смена оплаты)
```

### 5. Команда `/смены`

**Синтаксис:**
- `/смены` — за сегодня, по городам чата
- `/смены 25.02` — за конкретную дату
- `/смены барнаул` — по городу
- `/смены барнаул 25.02` — комбинация

**Формат вывода:**
```
📋 Смены оплаты: 25.02.2026

Барнаул Ленина — 2
• #12345 — 14:32 — 1 250 ₽
• #12389 — 18:15 — 980 ₽

Томск Центр — 1  
• #12401 — 20:05 — 2 100 ₽

Итого: 3 заказа
```

**Пустое состояние:**
```
📋 Смены оплаты: 25.02.2026

Нет заказов со сменой оплаты 👍
```
