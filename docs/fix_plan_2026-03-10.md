```sql
ALTER TABLE tenant_chats ADD COLUMN cities_json jsonb DEFAULT '[]'::jsonb;
UPDATE tenant_chats SET cities_json = jsonb_build_array(city) WHERE city IS NOT NULL AND city != '';
-- Не удаляем city сразу, оставляем для совместимости
```

- Убрать `city = data.cities[0] if data.cities else None`

**C. Там же** — `get_chats` (endpoint чтения чатов): убедиться что возвращает `cities_json` как `cities`

**D. `web/cabinet/connections.html`** — проверить что чекбоксы городов работают корректно с массивом

## Что НЕ входит в план

| Тема | Почему |
|------|--------|
| Убрать `tenant_id: int = 1` из 50+ функций | Дефолт не стреляет пока вызовы явные. Массовый рефакторинг без немедленной отдачи. |
| `verify=False` в iiko-клиентах | Документированный компромисс, сертификат iiko не проходит проверку — зафиксировано в `rules/integrator/iiko_api.md` |
| OLAP-слой: свести к одному клиенту | Архдолг, не ломает функциональность |
| JWT → HttpOnly cookie | Переписать весь auth-слой, overkill |
| `ctx_tenant_id` default=1 | Polling loop ставит контекст явно, риск только при добавлении новых handlers без контекста |
