# Web Spec — шаблон

> Этот файл заполняет @веб. Интегратор читает его перед реализацией — знает какой API делать и что отдавать.
> Имя файла: `название_страницы.md` (например: `dashboard_main.md`, `login.md`, `subscription.md`)

---

## Страница

**Название:** Главный дашборд

**URL:** `/dashboard`

**Тип:**
- [ ] Страница с данными (дашборд, отчёт)
- [ ] Форма (регистрация, оплата, настройки)
- [ ] Публичная (без авторизации)
- [ ] Приватная (нужен JWT / сессия)

---

## Авторизация

**Нужна:** да / нет

**Механизм:** JWT в cookie / Basic Auth / без

**Если нет токена:** редирект на `/login`

## Данные (что нужно от @интегратора)

### API endpoint

```
GET /api/stats
Query params:
  city: barnaul | abakan | tomsk | chernogorsk | all
  date: 2026-02-26 (ISO, опционально, дефолт: сегодня)

Response (JSON):
{
  "branches": [
    {
      "name": "Барнаул Основной",
      "city": "barnaul",
      "revenue": 87400,
      "orders_count": 43,
      "avg_check": 2033,
      "delays_count": 3,
      "avg_delivery_time": 48
    }
  ],
  "total": {
    "revenue": 141600,
    "orders_count": 70
  },
  "updated_at": "2026-02-26T14:32:00"
}
```

> Опиши точные поля — не «какие-то данные», а конкретные названия и типы.

## Компоненты страницы

### Header / фильтры

- Dropdown: выбор города (все / Барнаул / Абакан / Томск / Черногорск)
- Datepicker: выбор даты (дефолт: сегодня)
- Кнопка «Обновить»

### KPI-карточки (верхняя строка)

| Метрика | Источник | Формат |
|---|---|---|
| Выручка | `total.revenue` | `141 600 ₽` |
| Заказов | `total.orders_count` | `70` |
| Средний чек | `total.revenue / total.orders_count` | `2 023 ₽` |
| Опозданий | сумма `branches[].delays_count` | `3` |

### Таблица по точкам

Колонки: Точка / Выручка / Заказов / Ср. чек / Опоздания / Ср. время доставки

Сортировка по умолчанию: по выручке убывающая

### График (опционально)

- Тип: line chart
- Ось X: дни (последние 30)
- Ось Y: выручка
- Данные: отдельный endpoint `GET /api/stats/history?city=...&days=30`

## Пустые состояния

- Нет данных за день: карточки с `—`, таблица пустая, текст «Нет данных за выбранный период»
- Ошибка загрузки: toast «Не удалось загрузить данные. Попробуйте позже»
- Загрузка: skeleton-loader на карточках и таблице

## Стек и ограничения

**Фронтенд:** простой HTML+JS / React / Vue (выбрать при согласовании)

**Адаптивность:** десктоп обязательно, мобайл — желательно

**Браузеры:** Chrome, Safari (последние 2 версии)

**Дизайн-система:** нет (с нуля) / использовать Tailwind / использовать shadcn

## Тип страницы по аудитории

- [ ] **Публичная / продажная** — лендинг, тарифы, регистрация, оплата (аудитория: потенциальный клиент Аркентия)
- [ ] **Приватная / операционная** — дашборд, кабинет (аудитория: действующий клиент, авторизован)

Если публичная: думай как B2B-продажи — убеждение, ценность, простой следующий шаг.
Если приватная: думай как операционный инструмент — скорость, данные, минимум кликов.

## Мультиклиентность

- [ ] Страница показывает данные одного клиента (изолированы по tenant_id или отдельной БД)
- [ ] Страница общая для всех (лендинг, маркетинг)

## Заметки для @интегратора

> Что важно учесть при реализации бэкенда под эту страницу.

- CORS: разрешить только с домена дашборда
- Rate limit на `/api/stats`: не чаще 1 раз в 30 сек (данные не обновляются чаще)
- Кэш на сервере: ответ `GET /api/stats` кэшировать 30 сек
