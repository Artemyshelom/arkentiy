> Актуально для: `{bo_url}` = `https://{client}.iiko.it/resto`  
> Проверено живыми запросами на Ёбидоёби Абакан — 2026-03-09  
> Формат ответов: XML (если не указано иное)  
> Авторизация: все запросы требуют `?key={token}` в query params

## Авторизация

### `GET /api/auth`

Получить токен сессии.

**Параметры:**
- `login` — логин пользователя в iiko BO
- `pass` — SHA1-хэш пароля (hex, lowercase)

**Ответ:** UUID строкой, например `36d48b3a-2...`

**TTL токена:** ~1 час. После истечения — 401. Нужно рефрешить.

**Пример:**
```
GET /api/auth?login=admin&pass=da39a3ee5e6b4b0d3255bfef95601890
```

## Employees (Персонал)

### `GET /api/v2/employees`

Полный справочник сотрудников организации.

**Ответ:** список `<employee>` с полями:
| Поле | Описание |
|------|----------|
| `id` | UUID сотрудника |
| `name` | Полное имя |
| `firstName`, `lastName`, `middleName` | Имя по частям |
| `mainRoleId` | UUID основной должности |
| `mainRoleCode` | Код должности (`кур`, `ПСС`, `АС`, `АД` и т.д.) |
| `rolesIds` | Все роли (может быть несколько) |
| `roleCodes` | Коды всех ролей |
| `phone`, `cellPhone` | Телефоны |
| `preferredDepartmentCode` | Код привязанной точки |
| `departmentCodes` | Все точки ответственности |
| `hireDate`, `fireDate` | Даты найма/увольнения |
| `cardNumber` | Карта сотрудника (для учёта смен) |
| `deleted` | `true` если уволен/удалён |

**Размер:** ~20 МБ — общий справочник франчайзи, не только ваши точки.  
⚠️ Кешировать на 24 часа.

---

### `GET /api/v2/employees/roles`

Справочник всех должностей.

**Ответ:** список `<role>` с полями:
| Поле | Описание |
|------|----------|
| `id` | UUID роли |
| `code` | Краткий код (`кур`, `ПСС`, `АД` и т.д.) |
| `name` | Полное название |

**Количество:** ~314 ролей по всему франчайзи.

---

### `GET /api/v2/employees/salary`

Ставки всех сотрудников (история изменений).

**Ответ:** список `<salary>` с полями:
| Поле | Описание |
|------|----------|
| `employeeId` | UUID сотрудника |
| `dateFrom` | Дата начала действия ставки |
| `dateTo` | Дата окончания (`2500-01-01` = бессрочно) |
| `payment` | Ставка в **рублях за час** |

**Количество:** ~24 000 записей, из которых ~10 700 с `payment > 0`.  
⚠️ Кешировать на 24 часа.  
⚠️ Ставки всех сотрудников сети, не только ваших точек.

**Актуальная ставка на дату D:**
```
WHERE employeeId = X AND dateFrom <= D AND dateTo > D
```

---

### `GET /api/v2/employees/schedule`

Фактические смены сотрудников за период.

**Параметры:**
- `from` — дата начала, формат `YYYY-MM-DD` ⚠️ именно `from`, не `dateFrom`
- `to` — дата конца, формат `YYYY-MM-DD`

**Ответ:** список `<schedule>` с полями:
| Поле | Описание |
|------|----------|
| `id` | UUID смены (для дедупликации) |
| `employeeId` | UUID сотрудника |
| `roleId` | UUID роли в этой смене |
| `dateFrom` | Начало смены с TZ (например `2026-03-01T09:45:00+07:00`) |
| `dateTo` | Конец смены с TZ |
| `departmentId` | UUID точки |
| `departmentName` | Название точки (`Абакан_1 Кир`) |
| `scheduleTypeId` | UUID типа смены |
| `scheduleTypeCode` | Код типа (`день буд`, `ночь` и т.д.) |
| `nonPaidMinutes` | Неоплачиваемые минуты (обед и т.п.) |

**ФОТ-расчёт:**
```
worked_hours = (dateTo - dateFrom).seconds / 3600 - nonPaidMinutes / 60
fot = worked_hours × salary[employeeId].payment
```

⚠️ Данные только по точкам текущего `bo_url` (тенанта).

## Reports / OLAP

### `POST /api/v2/reports/olap`

Основной отчётный endpoint. Все аналитические запросы — через него.

**Content-Type:** `application/json`  
**Ответ:** JSON (в отличие от остальных — не XML)

**Тело запроса:**
```json
{
  "reportType": "SALES",
  "buildSummary": "false",
  "groupByRowFields": ["Department", "DishName"],
  "aggregateFields": ["DishSumInt.withoutVAT"],
  "filters": {
    "OpenDate.Typed": {
      "filterType": "DateRange",
      "periodType": "CUSTOM",
      "from": "2026-03-01",
      "to": "2026-03-02",
      "includeLow": "true",
      "includeHigh": "false"
    }
  }
}
```

---

### reportType: `SALES` — Продажи по блюдам

Обязательный фильтр: `OpenDate.Typed`

| Поле | Описание |
|------|----------|
| `Department` | Точка |
| `DishName` | Название блюда |
| `DishCode` | Код блюда |
| `DishCategory` | Категория |
| `DishSumInt.withoutVAT` | Выручка без НДС |
| `DishCount` | Количество |
| `WaiterName` | Имя курьера (доставка) |
| `OrderNum` | Номер заказа |
| `OpenDate.Typed` | Дата открытия |
| `CloseTime` | Время закрытия |
| `PayTypes` | Способы оплаты |
| `OrderDiscount.Type` | Тип скидки |
| `Storned` | Сторнировано (только для аудита) |
| `CashierName` | Кассир (только для аудита) |
| `DishDiscountSumInt.withoutVAT` | Сумма заказа (≠ скидка!) |
| `DiscountSum` | Сумма скидки ⚠️ корректна только в DELIVERIES |

### reportType: `DELIVERIES` — Данные доставки

Обязательный фильтр: `OpenDate.Typed`

| Поле | Описание |
|------|----------|
| `Delivery.Number` | Номер доставки |
| `Department` | Точка |
| `Delivery.CustomerPhone` | Телефон клиента |
| `Delivery.CustomerName` | Имя клиента |
| `Delivery.ActualTime` | Фактическое время доставки |
| `Delivery.ExpectedTime` | Обещанное время доставки |
| `Delivery.Address` | Адрес доставки |
| `Delivery.ServiceType` | Тип (доставка / самовывоз) |
| `Delivery.CancelCause` | Причина отмены |
| `Delivery.SourceKey` | Источник заказа (агрегатор) |
| `Delivery.PrintTime` | Время печати чека |
| `Delivery.CookingFinishTime` | Готово на кухне |
| `Delivery.SendTime` | Время отправки курьера |
| `OpenTime` | Время принятия заказа |
| `PayTypes` | Способы оплаты |
| `DiscountSum` | Сумма скидки ✅ корректна здесь |

---

### reportType: `TRANSACTIONS` — Финансовые транзакции (ДДС / ОПИУ)

Обязательный фильтр: `DateTime.DateTyped` (не `OpenDate.Typed`!)

| Поле | Описание |
|------|----------|
| `Account.Name` | Статья доходов/расходов |
| `Department` | Точка |
| `TransactionType` | Тип транзакции |
| `Amount` | Сумма (может быть отрицательной) |

Примеры `Account.Name` из реальных данных:
- `Выручка по доставке` — доход
- `Покупная стоимость продуктов` — COGS
- `Заработная плата линейного персонала (фикс.часть)` — ФОТ (агрегат)
- `Выдача зарплаты` — выплаты
- `Питание персонала` — расход
- `Комиссия по эквайрингу` — расход
- `Комиссия чиббис`, `Оплата яндекс.еда` — агрегаторы
- `Баннеры`, `ВКонтакте таргет`, `Реклама (еда)` — маркетинг

---

### reportType: `STOCK` — Складские движения

Обязательный фильтр: `EventDate` (не `OpenDate.Typed`!)

| Поле | Описание |
|------|----------|
| `ProductName` | Название позиции |
| `Department` | Точка/склад |
| `Amount` | Количество |

⚠️ Стоимостные поля (`TotalCost`, `Price`, `ProductCost`) — **не существуют** в этой версии API.  
⚠️ Приходные накладные через OLAP недоступны.

## Что НЕ доступно через API (проверено)

| Что хотели | Статус |
|-----------|--------|
| Приходные накладные с ценами | ❌ нет endpoint |
| Остатки на складе по стоимости | ❌ нет в OLAP STOCK |
| Прямой ФОТ по начислениям (зарплатный модуль) | ❌ нет endpoint |
| Справочник подразделений/точек | ❌ берётся из `employees.preferredDepartmentCode` или конфига |
| Справочник способов оплаты | ❌ нет |
| Лояльность / клиентская база | ❌ нет |
| Справочник складов | ❌ нет |

## Критические нюансы

1. **Мультитенантность:** каждый `bo_url` — отдельный сервер. Один запрос = данные одного тенанта. Итерировать по всем `bo_url` из `iiko_credentials`.
2. **Токен:** общий на тенанта, не на пользователя. Рефрешить при 401. Менеджер: `app/clients/iiko_auth.py`.
3. **`from`/`to` в schedule:** именно эти параметры, НЕ `dateFrom`/`dateTo` — вернёт 400.
4. **OLAP баг:** не включай `OpenDate.Typed` в `groupByRowFields` — обнуляет `Delivery.Number`.
5. **`DiscountSum`:** корректен только в `DELIVERIES`. В `SALES` считается per-dish — неправильный результат.
6. **SSL:** `httpx.AsyncClient(verify=False)` — сертификат iiko не проходит стандартную проверку.
