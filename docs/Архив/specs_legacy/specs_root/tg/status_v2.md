## Проблема текущей версии (v1)

При запросе нескольких точек бот отправляет N отдельных сообщений подряд — по одному на каждую точку. Пачка из 4-9 сообщений дезориентирует, чат засоряется.

## Сценарии

### Сценарий A — Одна точка

`/статус Барнаул_1` → одна точка → сразу **полная карточка** (текущий формат без изменений, + вариант А для времён, + кнопка "🔄 Обновить").

```
📍 Барнаул_1 Ана — 14:32
💰 Выручка: 87 400 ₽
🧾 Чеков: 43 | Средний чек: 2 033 ₽

💸 Скидки: 3 200 ₽ | SailPlay: 1 500 ₽
   └ Ручная скидка x 3: 1 200 ₽
   └ Промокод x 2: 800 ₽
📦 Себестоимость: 28.5%

🔴 Опозданий: 3 из 21 доставок (14.3%) | среднее: 12 мин
🕐 готовка 18 → ожидание 8 → в пути 22 мин
🚚 Заказы: 5 активных | доставлено: 21
   └ до отправки: 2 (готовится: 1, готовы: 1)
   └ в пути: 3
👥 На смене: поваров: 3, курьеров: 4

[ 🔄 Обновить ]
```

При нажатии "Обновить" → переопрашивает iiko → edit_message той же карточки.

## Формат строк в сводке

```
{branch_name_short}  {revenue}  {checks} чеков  {late_status}
```

Где `late_status`:
- `✅` — 0 опозданий
- `🟡 1 опозд.` — 1 опоздание
- `🔴 3 опозд.` — 2+ опозданий
- `⏳` — нет данных (RT не загружены)
- `⚠️` — ошибка получения данных

---

## Вариант А — строка времён в карточке

**Было:**
```
🕐 Готовка: 18 | Ожидание: 8 | В пути: 22 мин
```

**Стало:**
```
🕐 готовка 18 → ожидание 8 → в пути 22 мин
```

Стрелки `→` передают движение заказа по стадиям. Легче читается на телефоне.

---

## Кнопки — полная спецификация

| Кнопка | callback_data | Действие |
|--------|--------------|---------|
| `📍 Барнаул_1 Ана` | `stat:branch:{branch_name}` | edit_message: карточка точки |
| `🔄 Обновить` | `stat:refresh` | Переопросить все точки, edit_message: обновлённая сводка |
| `← Назад` | `stat:back` | edit_message: вернуться к сводке из кэша |
| `🔄 Обновить точку` | `stat:refresh:{branch_name}` | Переопросить одну точку, edit_message: обновлённая карточка |

> **branch_name** = полное название точки из конфига (например `Барнаул_1 Ана`).
> `stat:branch:Барнаул_1 Ана` = 26 байт — в пределах лимита 64 байт.

## Технические ограничения

- Длина сообщения: < 4096 символов
- `parse_mode`: `HTML`
- **Редактирование:** да — основной механизм навигации
- Запросы к iiko при обновлении: параллельные через `asyncio.gather`

## In-memory кэш

```python
_status_cache: dict[tuple[int, int], dict] = {}
# (chat_id, msg_id) -> {
#     "summary_text": str,
#     "summary_keyboard": list,
#     "branches_data": dict[str, dict],  # branch_name -> data от get_branch_status
#     "branches": list[dict],            # список branch-объектов для рефреша
# }
```

**TTL:** не нужен явный — при "Обновить" всё равно переопрашиваем. При перезапуске бота кэш сбрасывается — ок.

**Размер:** лимит 50 записей (FIFO).
