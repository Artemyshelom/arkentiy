# Архитектура системы доступа — Аркентий

> Управление правами чатов и пользователей без деплоя.
> Последнее обновление: 23 февраля 2026

---

## Иерархия

```
Город (Барнаул / Томск / Абакан / Черногорск / null = все)
  └── Чат (group chat_id)
        └── Модули (late_alerts, search, marketing, ...)
```

- Чат принадлежит ровно одному городу (или `null` = видит все города)
- Данные в командах автоматически фильтруются по городу чата
- Один чат может иметь любой набор модулей

---

## Модули и команды

| Модуль | Команды | Описание |
|--------|---------|----------|
| `late_alerts` | /опоздания (RT, сегодня) | Автоматические алерты опозданий |
| `late_queries` | /опоздания, /день | Ручные запросы истории |
| `search` | /поиск | Поиск заказа по номеру, телефону, блюду |
| `reports` | /статус, /повара, /курьеры | RT-отчёты по смене |
| `marketing` | /выгрузка | Выгрузка базы клиентов в CSV |
| `finance` | (зарезервировано) | Финансовые отчёты |
| `admin` | /конкуренты, /доступ | Служебные команды (только для admins) |

## Структура конфига `secrets/access_config.json`

```json
{
  "admins": [255968113],
  "chats": {
    "-1001000000001": {
      "name": "Опоздания Томск",
      "city": "Томск",
      "modules": ["late_alerts", "late_queries"]
    },
    "-1001000000002": {
      "name": "Маркетинг (все города)",
      "city": null,
      "modules": ["marketing"]
    },
    "-1001000000003": {
      "name": "Отчёты Барнаул",
      "city": "Барнаул",
      "modules": ["reports", "search"]
    }
  },
  "users": {
    "123456789": {
      "name": "РОП Иванов",
      "city": null,
      "modules": ["reports", "search", "late_queries"]
    }
  }
}
```

**Правила:**
- Файл в `secrets/` — не в git, не в .env
- Горячая перезагрузка — изменения применяются сразу, без деплоя
- Резервная копия создаётся автоматически при каждом изменении через бота

## Файлы системы

| Файл | Роль |
|------|------|
| `secrets/access_config.json` | Конфиг прав (не в git) |
| `app/access.py` | Загрузка конфига, проверка прав, hot-reload |
| `app/jobs/access_manager.py` | Telegram UI: /доступ + inline-кнопки + callbacks |
| `app/jobs/arkentiy.py` | Использует `app/access.py` вместо хардкода |

## Схема проверки прав при команде

```
Сообщение в чат/личку
        │
        ▼
get_permissions(chat_id, user_id)
        │
        ├─► admins[] → полный доступ (все модули, все города)
        │
        ├─► chats[chat_id] → модули + город чата
        │
        └─► users[user_id] → модули + город пользователя
                │
                ▼
        perms = {modules: set[str], city: str | None}
                │
                ▼
        cmd_module in perms.modules?  →  нет  →  молча игнорировать
                │ да
                ▼
        city фильтр автоматически в SQL/handler
```

## Telegram UI — управление доступом

### Главный экран `/доступ`

Только для admins (из `admins[]`). Открывается в личке.

```

БАРНАУЛ
  📍 Отчёты Барнаул  [⚙️]

ТОМСК

[➕ Добавить чат]  [👤 Пользователи]
```

## Обратная совместимость

Текущий `.env` остаётся как fallback:
- `TELEGRAM_ADMIN_ID` — всегда admin
- `TELEGRAM_ALLOWED_IDS` — работает если `access_config.json` не существует
- `TELEGRAM_MARKETING_IDS` — аналогично

После создания `access_config.json` — env-переменные игнорируются для проверки прав.
