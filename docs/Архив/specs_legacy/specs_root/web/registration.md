# Web Spec — Визард регистрации

> Статус: 🟡 На согласовании
> Автор: @веб
> Дата: 26 февраля 2026

---

## Страница

**Название:** Регистрация нового клиента (6-шаговый визард)

**URL:** `/register`

**Тип:**
- [x] Форма (регистрация)
- [x] Полу-публичная (без авторизации, создаёт аккаунт в процессе)

---

## Авторизация

**Нужна:** нет (визард создаёт аккаунт на последнем шаге)

---

## Аудитория

Потенциальный клиент, который уже видел лендинг и/или калькулятор и готов подключить Аркентия. Часто приходит с калькулятора с предзаполненной конфигурацией (URL-params).

**Главная задача страницы:** провести клиента через все шаги подключения — от регистрации до оплаты — и не потерять его по дороге.

---

## URL-параметры (из калькулятора)

Визард парсит query-параметры и предзаполняет данные:

```
/register?branches=3&cities=2&addons=finance,competitors&period=annual&promo=EARLY
/register?trial=true&branches=2&cities=1
```

| Параметр | Тип | Описание |
|----------|-----|----------|
| `branches` | int | Предвыбранное кол-во точек |
| `cities` | int | Предвыбранное кол-во городов |
| `addons` | string | Запятая-разделённые id модулей |
| `period` | string | `monthly` / `annual` |
| `promo` | string | Промокод для предзаполнения |
| `trial` | bool | `true` — запуск триала |

## Компоненты страницы

### Общая структура

```
[Header — тот же что на лендинге]

[Stepper — прогресс-бар шагов]
  1. Компания  ●────2. Города────3. Модули────4. iiko────5. Telegram────6. Обзор

[Footer: Назад / Далее]
```

**Поведение степпера:**
- Пройденные шаги — кликабельны, можно вернуться
- Текущий шаг — выделен (brand-600)
- Будущие шаги — серые, некликабельные
- На мобайле: показывать только «Шаг 2 из 6» текстом, без полного степпера

### Шаг 1 — Компания

```
Расскажите о вашем бизнесе

Название сети *
[________________________]

Контактное лицо *
[________________________]

Email *
[________________________]
  Будет использоваться для входа в кабинет

Телефон
[________________________]
  Необязательно

Пароль *
[________________________]
  Минимум 8 символов

                                    [Далее →]
```

**Валидация:**
- Название сети: обязательное, min 2 символа
- Контактное лицо: обязательное, min 2 символа
- Email: обязательный, формат email, проверка уникальности через API (`POST /api/auth/check-email`)
- Пароль: обязательный, min 8 символов
- Телефон: опционально, маска +7 (___) ___-__-__

**Валидация — в реальном времени** (при blur или debounce на ввод).

### Шаг 2 — Города и точки (пресеты из чейна)

```
Где работает ваша сеть?

Город 1
[▼ Выберите город        ]   [× Удалить]

  Точки в этом городе:
  [▼ Выберите точку             ]   [×]
  [▼ Выберите точку             ]   [×]
  [+ Добавить точку]

[+ Добавить город]

────────────────────────────────────
Выбрано: 2 точки в 1 городе

Нет вашего города? Напишите нам в Telegram

                      [← Назад]  [Далее →]
```

**Источник данных:** `/data/chain.json` — статический JSON, выгруженный из iiko BO (240 городов, 373 точки сети Ёбидоёби). Обновляется вручную по запросу.

**Поведение:**
- Dropdown города — список городов из chain.json; уже выбранные города скрыты из списка
- При выборе города — dropdown точек наполняется точками этого города
- Dropdown точки — точки выбранного города; уже выбранные скрыты
- Кнопка `+ Добавить точку` — новый dropdown с оставшимися точками (скрыта, если все выбраны)
- Кнопка `+ Добавить город` — новый блок (скрыта, если все города выбраны)
- Удаление точки/города — иконка `×` справа (не показывать если единственная)
- Если пришли с калькулятора (branches=3, cities=2) — создать пустые селекты
- Счётчик внизу обновляется в реальном времени
- Fallback: «Нет вашего города? Напишите нам в Telegram»

**Валидация:**
- Город и точка — обязательные (должны быть выбраны из списка)
- Дубликаты невозможны (уже выбранные скрыты из dropdown)

### Шаг 4 — Подключение iiko

```
Подключите вашу iiko

Для работы Аркентию нужен доступ к iiko BO.

URL вашего iiko BO *
[https://_______________:443]
  Обычно выглядит как https://your-server:443

Логин API iiko *
[________________________]
  Отдельный пользователь для API (не ваш личный логин!)

[Проверить подключение]

                      [← Назад]  [Далее →]
```

**Поведение кнопки «Проверить подключение»:**
- Состояние: idle → loading (спиннер) → success / error
- `POST /api/iiko/test-connection` с URL + логином
- Успех: зелёная галочка + «Подключение успешно! Найдено X точек»
- Ошибка: красный текст + конкретная причина:
  - `connection_refused` — «Не удаётся подключиться. Проверьте URL и доступность сервера»
  - `auth_failed` — «Неверный логин. Создайте отдельного API-пользователя в iiko»
  - `timeout` — «Сервер не отвечает. Попробуйте позже»
- Тест НЕ блокирует переход к следующему шагу — клиент может пропустить и настроить позже

**Подсказка (раскрывающийся блок):**
```
▸ Как создать API-пользователя в iiko?
  1. Откройте iiko BO → Администрирование → Пользователи
  2. Создайте нового пользователя с ролью "API"
  3. Скопируйте логин
```

```
Проверьте и оплатите

┌─ Ваш заказ ──────────────────────────────┐
│                                           │
│  Компания: Ёбидоёби                       │
│  Точки: Барнаул Основной, Б. Малиновка    │
│  Города: 1                                │
│  Модули: Базовый, Финансы                 │
│                                           │
│  ─────────────────────────────────────    │
│  Базовый      2 × 5 000       10 000 ₽   │
│  Финансы      2 × 2 000        4 000 ₽   │
│  ─────────────────────────────────────    │
│  Итого                        14 000 ₽/мес│
│                                           │
│  Подключение                  10 000 ₽    │
│                                           │
│  Промокод                                 │
│  [________________] [Применить]           │
│                                           │
│  ═════════════════════════════════════    │
│  К оплате сейчас:             24 000 ₽   │
│  (подключение + первый месяц)             │
└───────────────────────────────────────────┘

Период оплаты
(•) Ежемесячно    ( ) Ежегодно (-20%)

Способ оплаты
(•) Картой (ЮKassa)    ( ) Счёт для юрлица

[← Назад]  [Оплатить 24 000 ₽]

──── или ────

Попробуйте бесплатно 7 дней
Базовый пакет, данные за последний месяц
[Начать триал]
```

**Поведение:**
- Сводка данных из всех предыдущих шагов (только чтение, кликабельно для редактирования)
- Промокод: `POST /api/promo/validate` → пересчёт цены в реальном времени
- Промокод предзаполнен если пришёл из URL-params
- Переключение период → пересчёт итого
- «Картой» → `POST /api/payments/create` → редирект на ЮKassa
- «Счёт для юрлица» → показать поля ИНН, название юрлица → `POST /api/invoices/create` → скачивание PDF
- «Начать триал» → `POST /api/tenants/create` с `trial=true` → редирект в кабинет
- «Оплатить» → `POST /api/tenants/create` → `POST /api/payments/create` → редирект на ЮKassa
- Если промокод = free_connection, строка «Подключение» показывает зачёркнутую цену + «0 ₽»

## Сохранение прогресса

**Механизм:** `localStorage`

- При каждом нажатии «Далее» — данные текущего шага сохраняются в `localStorage('arkentiy_register')`
- При заходе на `/register` — проверка наличия сохранённых данных → предложение продолжить
- Формат: `{ step: 3, data: { company: {...}, cities: [...], modules: {...} }, timestamp }`
- Данные удаляются после успешной оплаты/создания триала
- Автоудаление через 7 дней (по timestamp)

**Banner при возвращении:**
```
┌──────────────────────────────────────────────────┐
│  У вас есть незавершённая регистрация             │
│  Компания: Ёбидоёби, шаг 3 из 6                  │
│  [Продолжить]  [Начать заново]                   │
└──────────────────────────────────────────────────┘
```

## Пустые состояния

- **Email занят:** жёлтое предупреждение под полем + ссылка «Войти»
- **iiko не подключается:** красный текст с конкретной ошибкой + кнопка «Пропустить, настрою позже»
- **Telegram не подключён:** аналогично iiko — кнопка «Пропустить»
- **Промокод невалиден:** красный текст «Промокод недействителен или истёк»
- **Ошибка создания тенанта:** toast «Произошла ошибка. Попробуйте позже» + retry кнопка
- **Ошибка оплаты:** редирект обратно на Шаг 6 + toast «Оплата не прошла. Попробуйте другую карту»

## Стек и ограничения

**Фронтенд:** HTML + Tailwind CSS + Vanilla JS

**Обоснование:** 6-шаговый визард — это по сути 6 форм, переключаемых по очереди. Состояние хранится в объекте JS и localStorage. Динамическая форма городов/точек — append/remove DOM-элементов. React избыточен для MVP — добавляет сложность сборки (Vite/Webpack) и деплоя. Vanilla JS с хорошей структурой справится.

**Адаптивность:** mobile-first
- На мобайле: 1 колонка, степпер свёрнут в текст
- На десктопе: форма по центру (max-width 640px), мини-сводка справа на Шаге 6

**Браузеры:** Chrome, Safari (последние 2 версии)

## Заметки для @интегратора

- URL-параметры из калькулятора — визард парсит и предзаполняет форму (Шаг 2 и 3)
- `POST /api/iiko/test-connection` НЕ сохраняет ничего — только проверяет доступность
- `POST /api/tenants/create` — основной вызов, создаёт всё: tenant, user, subscription, сохраняет iiko/telegram credentials
- При `trial=true` — не требует оплаты, сразу активирует тенант на 7 дней
- JWT-токен из `/api/tenants/create` — сохранять в cookie для последующего входа в кабинет
- Промокод может быть введён и в калькуляторе, и в визарде — бэкенд обрабатывает оба случая
- Rate limit: `/api/auth/check-email` — 10/мин/IP, `/api/iiko/test-connection` — 5/мин/IP
- После успешной оплаты ЮKassa redirect → `/payment/success?payment_id=...` → подтверждение + вход в кабинет
