# specs — артефакты дизайн-агентов

Папка с контрактами между дизайн-агентами и интегратором.

> **Роль файла:** правила ведения UX/Web спецификаций перед реализацией.
> **Единый путь:** `docs/specs/` (старый корневой `specs/` архивирован).
> **Когда обновлять:** при изменении процесса работы со spec-файлами или шаблонов.

## Структура

```
docs/specs/
  tg/        ← TG UX Specs от @ux-бот
    ШАБЛОН.md
    [название_функции].md
  web/       ← Web Specs от @веб
    ШАБЛОН.md
    [название_страницы].md
```

## Как использовать

**@ux-бот и @веб:**
1. Скопировать `ШАБЛОН.md` из нужной папки
2. Заполнить под конкретную задачу
3. Сохранить как `название_функции.md`
4. Сообщить @интегратору: «готов spec, смотри docs/specs/tg/название.md»

**@интегратор:**
1. Перед реализацией — прочитать spec
2. Если что-то неясно — уточнить у @ux-бот или @веб, не додумывать
3. После реализации — отметить в spec статус `✅ Реализовано`

## Статусы spec-файлов

Добавляй строку в начало файла:

```
> Статус: 🟡 В работе / ✅ Реализовано / 🔴 На доработке
```
