**URL:** `/login`

**Тип:**
- [x] Форма (авторизация)
- [x] Публичная (без авторизации)

---

## Авторизация

**Нужна:** нет (страница СОЗДАЁТ авторизацию)

**Механизм:** JWT в httpOnly cookie

**Если уже авторизован:** редирект на `/cabinet`

---

## Аудитория

Действующий клиент, который хочет войти в личный кабинет — посмотреть подписку, управлять модулями, оплатить.

**Главная задача страницы:** быстрый вход без лишних шагов.

## Компоненты страницы

### Форма входа

```
┌──────────────────────────────────┐
│                                  │
│          Аркентий                │
│    Вход в личный кабинет         │
│                                  │
│    Email                         │
│    [________________________]    │
│                                  │
│    Пароль                        │
│    [________________________]    │
│                                  │
│    [Войти]                       │
│                                  │
│    Забыли пароль?                │
│                                  │
│    ─────── или ───────           │
│                                  │
│    Нет аккаунта?                 │
│    Зарегистрируйтесь →          │
│                                  │
└──────────────────────────────────┘
```

**Поведение:**
- Email + пароль → `POST /api/auth/login`
- Успех → JWT в cookie → редирект на `/cabinet`
- Неверные данные → красный текст «Неверный email или пароль»
- Кнопка «Войти» — disabled пока не заполнены оба поля
- Enter на поле пароля → отправка формы
- «Забыли пароль?» → переход на `/forgot-password`
- «Зарегистрируйтесь» → `/register`

## Данные (что нужно от @интегратора)

### API endpoints

**1. Вход**
```
POST /api/auth/login
Body: { "email": "user@example.com", "password": "..." }

Response (200):
{
  "token": "jwt-...",
  "user": {
    "id": "uuid-...",
    "email": "user@example.com",
    "tenant_id": "uuid-...",
    "tenant_name": "Ёбидоёби"
  }
}
Set-Cookie: auth_token=jwt-...; HttpOnly; Secure; SameSite=Strict; Path=/

Response (401):
{ "error": "Неверный email или пароль" }
```

**2. Запрос сброса пароля**
```
POST /api/auth/forgot-password
Body: { "email": "user@example.com" }

Response (200): { "message": "ok" }
(всегда 200, даже если email не найден — безопасность)
```

**3. Сброс пароля**
```
POST /api/auth/reset-password
Body: { "token": "reset-token-...", "password": "new-password" }

Response (200): { "message": "Пароль изменён" }
Response (400): { "error": "Ссылка недействительна или истекла" }
```

## Пустые состояния

- **Неверные данные:** красный текст под формой «Неверный email или пароль»
- **Email не найден (forgot password):** всегда показывать «Письмо отправлено» (безопасность)
- **Токен сброса истёк:** «Ссылка истекла. Запросите новую» + ссылка на forgot-password
- **Уже авторизован:** автоматический редирект в кабинет

## Заметки для @интегратора

- JWT: exp 7 дней, refresh не нужен для MVP
- Cookie: httpOnly + Secure + SameSite=Strict
- Forgot password: генерировать одноразовый токен (UUID), TTL 1 час, хранить в БД
- Email: SMTP через Яндекс/Mail.ru (или SendPulse если нужны шаблоны) — настроить для support@arkentiy.ru
- Rate limit: `/api/auth/login` — 5 попыток/мин/IP, `/api/auth/forgot-password` — 3/мин/IP
- Не возвращать «email не найден» — всегда «письмо отправлено» (защита от enumeration)
