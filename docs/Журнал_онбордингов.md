## Метрики успеха онбординга

✅ **Клиент готов к production** когда:

1. **Data completeness** — все критические поля ≥ 90%
2. **Isolation test** — `SELECT DISTINCT tenant_id FROM orders_raw WHERE branch_name IN ('Веткa_1', 'Веткa_2')` = только его tenant_id
3. **Bot works** — `/статус`, `/отчёт`, `/поиск`, `/повара` возвращают ТОЛЬКО его данные
4. **No data leaks** — Ёбидоёби не видит его данные, он не видит Ёбидоёби
5. **Time metrics** — `cooking_duration`, `delivery_duration`, `total_duration` рассчитаны и имеют реальные значения

## Lessons Learned из Shaburov

**Обязательно перед следующим клиентом:**

1. ⚠️ **Multi-tenancy требует явного указания** — не дефолтить на tenant_id=1
2. ⚠️ **OLAP field names отличаются по версии iiko** — не копировать query, диагностировать
3. ⚠️ **Исторические данные — священное** — без них аналитика бесполезна, заполнять все phases
4. ⚠️ **Events API нужна stabilization period** — 24+ часа перед валидацией
5. ⚠️ **Role prefixes в iiko неполные** — расспросить клиента о локальных abbreviations (ПЗ, АЗ, КП и т.д.)
