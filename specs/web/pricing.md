## Аудитория

Потенциальный клиент, который уже понял что такое Аркентий (пришёл с лендинга) и хочет посчитать сколько это стоит для его сети.

**Главная задача страницы:** дать точную цену для конкретной конфигурации клиента и перевести на регистрацию.

## Данные (что нужно от @интегратора)

### Нет API-зависимостей

Все данные (цены, скидки) захардкожены на фронтенде. Калькулятор работает на чистом JS без обращений к серверу.

**Константы (в JS):**

```javascript
const PRICING = {
  base_per_branch: 5000,
  integration_fee: 10000,
  addons: {
    finance: { price_per_branch: 2000, label: "Финансы", description: "Сверка банковских выписок с кассой iiko" },
    competitors: { price_per_city: 1000, setup_per_city: 3000, label: "Конкуренты", description: "Мониторинг цен и меню конкурентов" }
  },
  volume_discounts: [
    { min: 4, max: 6, percent: 10 },
    { min: 7, max: 10, percent: 15 },
    { min: 11, max: null, percent: null, label: "Индивидуально" }
  ],
  annual_discount: 20,
  trial_days: 7
};
```

### Промокоды — нужен API

```
POST /api/promo/validate
Body: { "code": "WELCOME2026" }

Response (200):
{
  "valid": true,
  "code": "WELCOME2026",
  "bonuses": [
    { "type": "free_integration", "description": "Бесплатное подключение" },
    { "type": "fixed_discount", "amount": 2000, "months": 3, "description": "Скидка 2 000 ₽/мес на 3 месяца" }
  ]
}

Response (400):
{
  "valid": false,
  "error": "Промокод недействителен или истёк"
}
```

### 3. Калькулятор (основной блок)

Две колонки: слева — ввод параметров, справа — итоговая стоимость.

**Левая колонка — Параметры:**

```
Количество точек
[- ] 2 [ +]                          ← stepper, мин 1, макс 50

Количество городов
[- ] 1 [ +]                          ← stepper, мин 1, макс кол-во точек

Период оплаты
( ) Ежемесячно    (•) Ежегодно (-20%)  ← radio buttons

Дополнительные модули
[✓] Финансы — +2 000 ₽/точка/мес     ← checkbox
    Сверка банковских выписок с кассой

[✓] Конкуренты — +1 000 ₽/город/мес  ← checkbox
    Мониторинг цен в вашем городе
    + 3 000 ₽ разово за настройку каждого города

Промокод
[________________] [Применить]        ← текстовое поле + кнопка
✅ WELCOME2026 — бесплатное подключение, -2 000 ₽/мес на 3 мес
```

**Поведение калькулятора:**

- Пересчёт в реальном времени при любом изменении параметров
- Скидка за объём автоматически применяется и показывается отдельной строкой
- Volume discount: если точек >= 4 → строка "Скидка за объём (−10%)" появляется
- Годовая скидка: при выборе "Ежегодно" → строка "Скидка за год (−20%)"
- Скидки применяются к ежемесячной сумме последовательно (volume → annual → promo)
- При 11+ точках: вместо расчёта показать "Свяжитесь с нами для индивидуального предложения" + контактные данные
- Промокод: кнопка "Применить" → запрос к API → если валиден, показать бонусы + пересчитать
- CTA "Подключить" → `/register?branches=2&cities=1&addons=finance,competitors&period=annual&promo=WELCOME2026`
- CTA "Начать триал" → `/register?trial=true&branches=2&cities=1`

### 5. FAQ (аккордеон)

```
▸ Как считается стоимость?
  Базовый пакет × количество точек + дополнительные модули.
  Скидки применяются к итоговой сумме.

▸ Что входит в бесплатный триал?
  7 дней базового пакета. Бэкфил данных за последний месяц — 
  вы сразу увидите реальные отчёты и алерты. Если не понравится — 
  данные удаляются через 7 дней после окончания триала.

▸ Можно ли оплатить по счёту?
  Да, выставляем счёт для юридических лиц с актом.

▸ Что если у меня больше 10 точек?
  Свяжитесь с нами — предложим индивидуальные условия.

▸ Как добавить модуль после подключения?
  В личном кабинете в один клик. Стоимость пересчитывается пропорционально.

▸ Нужен ли iiko для работы?
  Да, Аркентий работает с iiko BO. Поддерживаются все версии iiko с API.
```

## Пустые состояния

- **Промокод невалиден:** текст под полем в красном: "Промокод недействителен или истёк"
- **Промокод валиден:** текст под полем в зелёном: "✅ [описание бонусов]"
- **11+ точек:** вместо итоговой суммы: "Индивидуальное предложение — свяжитесь с нами"
- **Ошибка API (промокод):** "Не удалось проверить промокод. Попробуйте позже"

## Стек и ограничения

**Фронтенд:** HTML + Tailwind CSS + Vanilla JS

**Обоснование:** калькулятор — это JS-логика над статическими данными. Единственный API-вызов — проверка промокода. React избыточен для одной страницы с калькулятором. Все вычисления на клиенте.

**Адаптивность:** mobile-first
- На мобайле: параметры сверху, итог снизу (не sticky)
- На десктопе: 2 колонки, итог sticky справа

**Браузеры:** Chrome, Safari (последние 2 версии)

## Заметки для @интегратора

- Цены захардкожены в JS — при изменении тарифов нужно обновить HTML-файл
- В будущем можно перенести цены в API (`GET /api/pricing`), но на MVP это overkill
- URL-параметры при переходе на `/register` — визард должен парсить и предзаполнять форму
- Промокод можно ввести и на калькуляторе, и в визарде — бэкенд должен обрабатывать оба случая
- Rate limit на `/api/promo/validate`: 10 запросов в минуту с одного IP (защита от перебора)
