# Правила для Cursor Agent в проекте Аркентий

## 🎯 Основной принцип
Придерживаться структуры проекта. Не разбрасывай информацию по разным местам.

## 📁 Где что живёт
- **Архитектура** → `rules/integrator/architecture.md` (не в PROJECT_UPDATE_STATUS)
- **База данных** → `rules/integrator/database.md`
- **Lessons learned** → `rules/integrator/lessons.md`
- **Статус сессии** → `PROJECT_UPDATE_STATUS.md` (только статус, без архитектуры)
- **История работы** → `docs/journal.md`

## 🔧 Перед редактированием файла
1. Прочитай весь файл или минимум контекст (20 строк до и после)
2. Используй `multi_replace_string_in_file` для параллельных правок
3. Не предлагай, а делай — имплементируй прямо

## ✅ Код
- **Python 3.11+**, только async functions (`async def`, `await`)
- Обязательна типизация: `def func(param: str) -> dict[str, Any]:`
- Docstrings на русском (краткие)
- Логирование: `logger.info/warning/error`, не `print()`
- Ловить конкретные exceptions, не голый `except:`

## 🗂 Git workflow
- Feature branches: `feat/название`
- Bugfix: `fix/название`
- Коммиты на русском: `feat: добавил сверку ТБанк`
- Перед пушем: локално проверить (`docker compose up --build`)

## 🚫 Антипаттерны (из lessons.md)
- Не используй iiko auth напрямую — только через `iiko_auth.py` мага
- UPSERT в БД, не UPDATE или INSERT отдельно
- Индексы на `tenant_id`, `branch_id`, `created_at`
- ContextVar `_ctx_tenant_id` для текущего тенанта в запросе (не передавай через параметры везде)

## 📝 Документирование
- Архитектурные решения → `rules/integrator/`
- Найденные баги → `rules/integrator/lessons.md`
- История сессии → `docs/journal.md` в конце
- Не дублируй информацию в разных местах

## 🔐 Секреты и конфиги
- `/opt/ebidoebi/secrets/` на VPS (НЕ в git)
- `bank_accounts.json` (per-tenant конфиг)
- `api_keys.json` (iiko, TG, Sheets)
- `.env` переменные → `config.py`
