# API Аркентия для Бориса — Справочник

> Инструкция для AI-агента Бориса по работе с аналитическим API.
>
> **Base URL:** `https://arkenty.ru/api/stats`  
> **Auth:** `Authorization: Bearer <token>`  
> **Rate limit:** 60 запросов/мин на токен

---

## Как выбрать нужный endpoint

| Задача | metric | Когда использовать |
|--------|--------|--------------------|
| Что творится прямо сейчас | `realtime` | Активные заказы, текущие опоздания |
| Итоги конкретного дня | `daily` | Сводные KPI за день |
| Тренд за период | `period` | Неделя, месяц, сравнение периодов |
| Кто работал в смену | `shifts` | Состав персонала за день |
| **Почасовой разрез** | **`hourly`** | **Анализ нагрузки, поиск причин опозданий** |

---

## metric=hourly — Почасовая аналитика

**Для чего:** глубокий анализ нагрузки по часам — когда пиковая нагрузка, почему были опоздания в конкретный час, хватало ли персонала.

### Запрос

```
GET /api/stats?metric=hourly&date=YYYY-MM-DD[&branch=название][&city=город]
```

**Параметры:**
- `date` — дата (YYYY-MM-DD). По умолчанию: вчера.
- `branch` — фильтр по названию точки (подстрока, регистронезависимо)
- `city` — фильтр по городу

### Ответ

```json
{
  "date": "2026-03-07",
  "branches": [
    {
      "name": "Барнаул Попова 34",
      "city": "Барнаул",
      "hours": [
        {
          "hour": "2026-03-07T03:00:00+00:00",
          "orders_count": 3,
          "revenue": 2700,
          "avg_check": 900,
          "avg_cook_time": 18.5,
          "avg_courier_wait": 4.2,
          "avg_delivery_time": 52.1,
          "late_count": 1,
          "late_percent": 33.3,
          "cooks_on_shift": 2,
          "couriers_on_shift": 1,
          "orders_in_progress": 5
        }
      ]
    }
  ]
}
```

### Поля объяснения

| Поле | Тип | Значение | Примечание |
|------|-----|----------|------------|
| `hour` | ISO datetime | Начало часа (UTC) | Для перевода в локальное время: UTC+7 для всех точек Артемия |
| `orders_count` | int | Заказов **завершено** за этот час | Только `Доставлена`/`Закрыта`, без отменённых |
| `revenue` | int | Выручка за час (руб.) | Сумма по завершённым заказам |
| `avg_check` | int | Средний чек (руб.) | |
| `avg_cook_time` | float/null | Среднее время готовки (мин) | `cooked_time - opened_at`. NULL если OLAP ещё не пришёл (данные за сегодня) |
| `avg_courier_wait` | float/null | Ожидание курьера (мин) | `ready_time - cooked_time` |
| `avg_delivery_time` | float/null | Итоговое время доставки (мин) | `actual_time - opened_at` (от приёма до вручения) |
| `late_count` | int | Заказов с опозданием | Все типы заказов |
| `late_percent` | float | % опозданий от завершённых | |
| `cooks_on_shift` | int | Поваров на смене в этот час | Пересечение смены с временным диапазоном часа |
| `couriers_on_shift` | int | Курьеров на смене в этот час | Аналогично |
| `orders_in_progress` | int | Заказов в работе на **начало** часа | Накопленная очередь из предыдущих часов |

### Важные оговорки

- **Тайминги NULL за сегодня** — OLAP enrichment (тайминги) приходит утром ~05:26 МСК за вчера. За текущий день `avg_cook_time` и др. будут `null`, пересчёт в 06:35 МСК.
- **`orders_in_progress`** — заказы, открытые до начала часа и ещё не завершённые. Например, в 14:00 показывает заказы, которые приняты до 14:00 и до сих пор не доставлены. Полезно для понимания очереди.
- **Час в UTC** — локальное время = `hour` + 7 часов (UTC+7 для всех точек Артемия, UTC+6 для Абакана / точек Шабурова — уточняй если нужно).
- **Самовывоз включён** — данные включают все заказы, в т.ч. самовывоз. При анализе "загруженности доставки" учитывай это.

### Примеры запросов

```bash
# Вчера по всем точкам
curl "https://arkenty.ru/api/stats?metric=hourly" \
  -H "Authorization: Bearer TOKEN"

# Конкретный день, конкретная точка
curl "https://arkenty.ru/api/stats?metric=hourly&date=2026-03-07&branch=Барнаул" \
  -H "Authorization: Bearer TOKEN"

# Весь город
curl "https://arkenty.ru/api/stats?metric=hourly&date=2026-03-07&city=Барнаул" \
  -H "Authorization: Bearer TOKEN"
```

### Типичные аналитические запросы

**Пиковые часы нагрузки:**
→ Отсортировать hours по `orders_count` убыванию.

**Корреляция "мало курьеров → опоздания":**
→ Найти часы где `couriers_on_shift` = 1 и `late_percent` > 30%.

**Кухонный bottleneck:**
→ Часы где `avg_cook_time` > 30 мин — вероятна перегрузка кухни.

**Накопленная очередь:**
→ `orders_in_progress` в начале часа + `orders_count` за час = суммарная нагрузка.

## metric=daily — Итоги дня

```
GET /api/stats?metric=daily[&date=YYYY-MM-DD][&branch=...][&city=...]
```

Возвращает сводные KPI за день по каждой точке: выручка, чеки, средний чек, % опозданий, тайминги, новые/постоянные клиенты.

По умолчанию `date` = вчера.

---

## metric=period — Агрегат за период

```
GET /api/stats?metric=period&from=YYYY-MM-DD&to=YYYY-MM-DD[&branch=...][&city=...]
```

Суммирует daily_stats за период. Тайминги взвешенные по кол-ву заказов.

---

## metric=realtime — Текущее состояние

```
GET /api/stats?metric=realtime[&branch=...][&city=...]
```

Активные заказы прямо сейчас, текущие опоздания, RT-тайминги из Events API iiko. Без параметра `date`.

---

## metric=shifts — Смены

```
GET /api/stats?metric=shifts[&date=YYYY-MM-DD][&branch=...][&city=...]
```

Список сотрудников с ролями за день. По умолчанию `date` = сегодня.
